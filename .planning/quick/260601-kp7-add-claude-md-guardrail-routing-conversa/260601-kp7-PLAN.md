# Quick Task 260601-kp7 — CLAUDE.md routing guardrail + WR-04 tracking record

## Objective

Two coupled outcomes from the same observation: during the Phase 3 follow-up triage, a
conversational "yes please fix that" turned into direct edits — including a real **code**
change (the WR-04 PII-logging hardening) — applied outside any GSD workflow wrapper, so it
carries no PLAN/SUMMARY/verify trail.

1. Add a governance guardrail to `CLAUDE.md` (GSD Workflow Enforcement) that tells the agent
   to recognise when a conversation turns into a *file change* and route it to the correct
   `/gsd` command instead of silently editing — with a clear class-based rule (code vs docs vs
   trivial) and a stricter definition of "explicit bypass".
2. Retroactively record the already-applied WR-04 change (commit `1c5b347`) so it has a tracked
   artifact. The code itself is done and green; this task only documents and links it.
