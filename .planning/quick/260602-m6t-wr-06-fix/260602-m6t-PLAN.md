<objective>
Fix WR-06 (Phase 05 review): in registrar.py _handle, `try: body = resp.json()
except Exception: body = {}` uses a blanket `except Exception`. The intent (per the
comment) is only to tolerate a non-JSON body, but the bare clause also swallows
unrelated failures (e.g. a TypeError/AttributeError from an unexpected response
object), masking genuine programming errors as "empty body → fall through to status
checks."

Fix: catch only the decode error so unrelated failures still surface.

Purpose: stop masking real bugs behind the non-JSON-body tolerance.
Output: narrowed except in _handle + regression tests.
</objective>
