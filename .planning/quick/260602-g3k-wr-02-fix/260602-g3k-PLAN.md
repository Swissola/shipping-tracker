<objective>
Fix WR-02 (Phase 05 review): `os.makedirs(os.path.dirname(db_path) or "data", exist_ok=True)`
at shipping_tracker/main.py:80 creates the WRONG directory for bare-filename paths.

When DATABASE_PATH has no directory component (e.g. `tracker.db`), `os.path.dirname`
returns `""`, so the `or "data"` fallback fabricates a `data/` directory — but
`sqlite3.connect(db_path)` then writes `tracker.db` to the CWD, not `data/`. Result:
a spurious empty `data/` dir plus a DB file in an unexpected location. For
`DATABASE_PATH=:memory:` it needlessly creates `data/` on disk for an in-memory DB.

Fix: only create a directory when the path actually has one, so the makedirs target
always matches what `connect()` uses.

Purpose: Eliminate the mismatched/spurious directory side effect.
Output: Corrected dir-creation in main(), plus a regression test.
</objective>
