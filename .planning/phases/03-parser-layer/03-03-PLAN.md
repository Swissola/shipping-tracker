## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| list[RawEmail] (untrusted email content) → dispatch loop | Each email is attacker-influenceable content from the inbox |
| Dispatch loop → log file (public-repo project) | Any field logged here risks PII in committed/shared logs |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-08 | Denial of Service | main() dispatch loop | mitigate | A no-tracking (extract -> None) or no-match email is logged and skipped, never raised — one bad email cannot crash the run (D-05, PARSE-03); verified by test_dispatch_no_match_skips + test_dispatch_preshipment_skips |
| T-03-09 | Information Disclosure | main() dispatch log calls | mitigate | All dispatch log calls carry only message_id / counts (`%`-style); body/sender/tracking_number never logged (LOG-02); grep acceptance criterion enforces this |
| T-03-10 | Tampering | _get_all_sender_domains() vs GMAIL_SENDER_LIST drift | mitigate | Sender list is derived from parser-owned constants (single source of truth), eliminating the env/parser divergence that would silently fetch zero or wrong-sender emails (D-01, RESEARCH.md Pitfall 5) |
| T-03-SC | Tampering | npm/pip/cargo installs | accept | No packages installed in Phase 3 (RESEARCH.md Package Legitimacy Audit: stdlib only); no install task to gate |
