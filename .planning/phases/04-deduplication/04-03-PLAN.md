<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| email.message_id / tracking_number -> SQL in the loop | The D-03 INSERT OR IGNORE in main() must be parameterized |
| DATABASE_PATH (.env) -> filesystem | An attacker-controlled path could traverse directories or hit a privileged location |
| per-email exception -> log file | A raising parser/registrar must not leak email content into logs (WR-04 / LOG-02) |
| DB file on the Pi -> other local users | SQLite file holds real tracking numbers + opaque Gmail ids |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-08 | Tampering | D-03 INSERT OR IGNORE in main() loop | mitigate | Parameterized `?` placeholders for message_id + timestamp (ASVS V5) |
| T-04-09 | Information disclosure | WR-04 per-email error log | mitigate | Reuse the established pattern: log `id=%s type=%s` with type(exc).__name__ only; no logger.exception, no body/tracking number (LOG-02) |
| T-04-10 | Tampering / EoP | DATABASE_PATH directory traversal or unexpected write location | accept (low) | Single-user local cron tool; DATABASE_PATH is operator-set in their own .env, not attacker input. Documented in .env.example; `os.path.dirname(...) or "data"` guard prevents empty-path crash. No traversal-from-untrusted-source path exists |
| T-04-11 | Information disclosure | SQLite file readable by other Pi users | accept (low) | `*.db` gitignored so it never reaches the public repo; file lives under the operator's own data dir. POSIX umask governs local perms on a single-user Pi — out of scope to harden further for v1 (documented honestly) |
| T-04-12 | Denial of service | Cron overlap causing "database is locked" mid-run | accept (low) | PRAGMA busy_timeout=5000 (set by init_db, D-10) waits; INSERT OR IGNORE makes the D-03 write idempotent under overlap (pitfall 4) |
| T-04-SC | Tampering | npm/pip/cargo installs | n/a | Zero new dependencies — stdlib sqlite3/datetime + existing python-dotenv (RESEARCH §Package Legitimacy Audit: not applicable) |
</threat_model>
