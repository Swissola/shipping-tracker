<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| tracking_number / message_id -> SQL | Values flow into SQLite queries; must be parameterized, never string-formatted |
| db.py / registrar.py -> log file | Operational data (tracking numbers) must never reach logs (LOG-02) |
| registrar callable -> register_and_persist | An injected callable may raise; its exception message could carry PII if careless |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-04 | Tampering | All db.py queries (SQL injection via tracking_number / message_id) | mitigate | Every query uses parameterized `?` placeholders; acceptance criteria forbid f-strings/concatenation (ASVS V5, RESEARCH §Security Domain) |
| T-04-05 | Information disclosure | db.py / registrar.py logging | mitigate | LOG-02: log only message_id (opaque, non-PII) + counts; never tracking_number; NullRegistrar logs no value; no logger.exception (no traceback) |
| T-04-06 | Information disclosure | registrar exception message reaching a log | mitigate | register_and_persist re-raises to main.py WR-04 which logs type(exc).__name__ only — never the exception string/traceback (Plan 03 owns the log site) |
| T-04-07 | Denial of service | DB lock under brief cron overlap | accept (low) | PRAGMA busy_timeout=5000 waits rather than instantly raising "database is locked" (D-10); single-writer cron tool — WAL rejected as overkill |
| T-04-SC | Tampering | npm/pip/cargo installs | n/a | Zero new dependencies — stdlib sqlite3/datetime/typing only (RESEARCH §Package Legitimacy Audit: not applicable) |
</threat_model>
